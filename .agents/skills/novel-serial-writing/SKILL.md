# 长篇连载写作 Skill

## 最简一行版

```
看纲 → 看状态台账 → 填章节记录 → 写正文 → 勾第10节检查 → 回填台账 → 按触发补POV/能力/阶段/势力/规则/地图/卷尾/运营/角色/实体文件
```

## 适用场景

- 写某一章正文
- 填/改章节记录
- 写后连续性检查与台账回填
- 开新书初始化文档结构
- 每10章小复盘 / 每卷大复盘

**禁止**：跳过章节记录直接写正文；写完不同步台账。

---

## 1. 看纲定层级

按顺序定位本章功能，不要跳层：

```
13_整书总纲.md
  → 总纲.md
  → 卷纲（卷XX_卷名/章纲/卷XX_卷纲.md）
  → 阶段大纲（卷XX_阶段XX_XXX-XXX章大纲.md）
  → 当前10章章纲（卷XX_XXX-XXX章章纲.md）
```

若当前10章章纲缺失，先补章纲，再写正文。

规则主文件：`00_写作约束与流程.md`

## 3. 先写章节记录

复制 `01_章节记录模板.md` 到 `卷XX_卷名/章节记录/第XXX章_章节记录.md`。

**顺序**：

1. 先填第2节「六问」：
   - 推进哪一层主线目标？
   - 核心冲突是什么？
   - 结束后局面怎么变？
   - 读者对谁产生新认知？
   - 有没有新物品/能力/规则/伏笔？
   - 章末钩子是什么？
2. 六问答不出 → 不写正文
3. 再补完整份记录（第1–9节）

---

## 4. 再写正文

保存到 `卷XX_卷名/正文/第XXX章_标题.md`。

写中优先保证：**场景清晰、冲突明确、结尾有钩子**。

硬约束：

| 项 | 规则 |
|----|------|
| 字数 | 硬下限2000字；软目标2000–2200字；必要时可上浮至2300字 |
| 新信息 | 单章最多1个关键新信息 |
| 章节功能 | 至少完成「剧情 / 关系 / 信息 / 伏笔」中的1项 |
| 主线 | 至少挂住主线四层中的1层 |
| 解释句 | `明白了/看懂了/意识到/这说明` 每章最多1处 |
| 章末排比 | 每3章最多1次 |
| 开篇节奏 | 每5章至少1次变体 |

---

## 5. 写完先做连续性检查

先把章节记录**第10节**逐项勾完：

- [ ] 字数（不少于2000字，软目标2000–2200字，必要时不超过2300字）
- [ ] 时间线
- [ ] 视角
- [ ] 设定冲突
- [ ] 能力代价
- [ ] 伏笔登记
- [ ] 是否服务卷目标
- [ ] 文风是否跑偏
- [ ] 势力/组织权限是否变更
- [ ] 异常、门域或规则场是否新增
- [ ] 地点、路线或空间层级是否新增
- [ ] 卷尾必须回收项是否推进
- [ ] 简介、封面或运营承诺是否受影响
- [ ] 角色是否退场、暂退或再登场
- [ ] 怪物、异常实体或怪潮可视化是否新增

未勾完 → 先修正文或记录，再回填台账。
