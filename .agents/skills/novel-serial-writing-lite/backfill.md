# lite 回填台账怎么做

原则：**写章卡里已经写好了「本章变更」；回填 = 把变更抄进台账行，不是重新读全书再想一遍。**

---

## 三档回填（按成本）

| 档 | 何时 | 做什么 | 读多少 |
|----|------|--------|--------|
| **每章必回** | 每章填字后立刻 | 只追加，不通读 | 用 Shell/编辑器在文件**末尾追加**；或 Grep 表头后只 Read 末 20 行确认格式 |
| **有变才回** | 写章卡里标了变化 | 改对应条目的「当前状态」一行 | Grep 人名/伏笔ID/物品名 → 只改那一段 |
| **节点对账** | 每 10 章 / 用户说「对账」 | 用上下文包 vs 台账抽查 | 可多读一点，单独开会话 |

不要攒着「写完 10 章再回填」——`02/08` 一断，下一章建纲会对不齐。

---

## 每章必回（从写章卡抄）

写章卡 → 台账字段映射：

| 写章卡 | 写入 |
|--------|------|
| 标题、功能、局面变化、钩子、字数 | `02_章节总表.md` **新一行** |
| 时间、地点、事件一句 | `08` 时间轴 **新一行** |
| 未收钩 / 禁触 / 现在时 | `当前写章上下文包.md` **覆盖刷新**（短） |

操作口令（给 Agent）：

```
不通读02/08全文。Read末15行对齐表格式，然后追加本章一行。
```

---

## 有变才回（写章卡先打勾）

建卡/补卡时在章末加一行（可选但推荐）：

```markdown
### 回填标记
- 03人物：无 / 有→【谁：改成什么】
- 04关系：无 / 有→【谁与谁】
- 05物品能力：无 / 有→【名称】
- 06伏笔：新埋伏笔ID? / 触动? / 回收?
- 14-20：无 / 有→【文件号+一句】
```

填字后 Agent **只处理「有」的项**：

1. `Grep` 关键词定位旧行  
2. 改「当前状态」或追加新行  
3. **禁止**为回填打开整本 `03/05`

无变化的文件：**零字节读取**。

---

## 章节记录怎么回填

lite 允许写后薄落地，仍建议有文件，方便原版流程以后接上：

- 从写章卡复制：定位、六问、节拍、在场  
- 补：实际字数、第10节勾选  
- 不要再写一篇「第二份剧情」

---

## 和「上下文会爆」的关系

| 错误做法 | 正确做法 |
|----------|----------|
| 回填前把 03+05+08 全文读进对话 | 末尾追加 / Grep 改一行 |
| 每章「检查一下所有台账是否一致」 | 一致性留给每10章对账会话 |
| 回填时重新推理设定 | 只抄写章卡「回填标记」 |

回填本身几乎不占上下文，**占上下文的是通读**。

---

## 每10章对账会话（单独开）

用户口令：`用 lite 对账 第121-130`

1. 读上下文包 + 这段10章章纲目录  
2. Grep `02` 是否 10 行都在  
3. 抽查 `06` 未回收是否与包内钩子一致  
4. 修偏差，刷新上下文包  
5. **不写正文**

---

## 一句话决策

- **日常**：卡上的变更 → 追加 `02/08` + 刷包 + 有变才 Grep 改 `03–06`  
- **怕漏**：每10章开一次对账会话，不要每章全量扫
