# 《全球迷雾：我能校正万物词条》写作约束与流程

## 目标

这份文档的作用，不是限制灵感，而是防止长篇连载在中后期出现以下问题：

- 主线散掉
- 人物动机跑偏
- 设定越写越乱
- 能力无限膨胀
- 伏笔埋了不回收
- 写到后面忘了前面发生过什么

## 一、硬约束

### 1. 主线优先级

本书所有剧情都必须服务以下四层目标：

1. 表层目标：活下去，保护妹妹
2. 中层目标：查清父亲失踪真相，并确认他的立场
3. 深层目标：查清沈辞为什么会成为“编目师”
4. 终极目标：决定人类是否接受被编目、被管理的命运

如果一段剧情和这四层目标都无关，就不要写，或者必须压缩。

### 2. 世界观来源统一

本书所有超常现象，原则上都要能回到以下体系：

- 灰雾
- 门域
- 失名
- 序列
- 编目庭

不要额外引入一套完全无关的新神秘体系。  
新设定可以扩展，但不能脱离现有底层逻辑。

### 3. 主角写法约束

沈辞的核心竞争力不是纯战力，而是：

- 观察
- 判断
- 识别规则
- 校正词条
- 在混乱中做出正确选择

沈辞可以变强，但不能过早写成“单纯靠硬打横推一切”的角色。  
他的强，应该更多体现在看得更深、改得更准、承担得更多。

### 4. 能力写法约束

每一种重要能力都必须明确：

- 触发条件
- 能力效果
- 使用代价
- 当前上限
- 已知风险

禁止出现“临时为了救场突然多出一个能力”的写法。  
如果能力升级，必须有铺垫、有代价、有记录。

### 5. 人物写法约束

每个重要角色都至少承担一种核心功能：

- 推主线
- 做对照
- 制造冲突
- 提供情感锚点
- 揭示世界观

如果一个角色连续多章既不推动剧情，也不改变关系，也不补充信息，就要考虑合并、删减或让其退场。

### 6. 信息投放约束

每一章最多只扩展一种关键新信息：

- 一个新规则
- 一个新势力
- 一个新能力
- 一个关键真相碎片

不要在单章内同时塞太多世界观解释，否则节奏会发闷。

### 7. 章节功能约束

每一章至少要完成以下四项中的一项，最好完成两项：

- 推进剧情
- 推进人物关系
- 揭示新规则或新信息
- 埋下或回收伏笔

如果一章四项都没有完成，那就是无效章节。

### 8. 章节字数约束

每章正文必须控制在 `2000-2100字` 之间。

推荐执行方式：

- 目标字数：2050字左右
- 下限：2000字
- 上限：2100字

这个约束的目的，是保证连载节奏稳定，避免有的章节太散，有的章节又过长失控。

如果一章写不到2000字，通常说明：

- 冲突不够
- 信息量不足
- 章节功能不完整

如果一章超过2100字，通常说明：

- 事件塞太多
- 解释太多
- 节奏拖慢

超出范围时，优先拆分章节，而不是继续硬塞。

### 9. 爽点与悬念平衡

本书是长篇商业向小说，不能只有设定和气氛，还要持续给读者反馈：

- 小爽点：能力初显、机智反杀、资源到手
- 中爽点：规则破解、身份抬升、打脸翻盘
- 大爽点：封门、立城、改写秩序、真相炸开

同时每个阶段都必须保留悬念，不要一次性把谜底讲完。

### 10. 文风去模板化约束

为了避免连载写到中段后出现明显套路感，正文必须额外遵守以下五条：

1. 章末短句排比收尾，每3章最多使用1次  
   其余章节优先用以下方式收尾：
   - 画面定格
   - 动作未完成
   - 新物件露头
   - 冷信息落地

2. 总结式旁白要克制  
   每章`明白了 / 看懂了 / 意识到 / 这说明 / 也就是说`这类解释句，原则上最多保留1处。  
   能让物件、规则、动作自己说话的地方，就不要再由旁白重复解释。

3. 新场景开篇要轮换写法  
   不要每次都写成“环境描写 + 感官细节 + 氛围铺垫”的固定三段式。  
   开篇至少轮换以下几种方式：
   - 直接动作切入
   - 先看见规则牌
   - 先看见异常物
   - 先听见声音或播报

4. 配角必须有语气区分  
   关键配角不能共用同一种冷静、简洁、书面化口吻。  
   至少要给主要配角固定一个可识别的小特征，比如：
   - 林照：短句、判断先行、偶尔带一点硬气
   - 执行层角色：重复、机械、像流程在开口
   - 情绪型配角：说话更碎、更乱、更像活人反应

5. 章节结构不能过于均匀  
   不要长期稳定重复“场景描写 → 规则展示 → 线索推进 → 结尾钩子”四件套。  
   每5章至少主动安排1章节奏变体，比如：
   - 偏动作章
   - 偏信息章
   - 偏潜行章
   - 偏情绪压迫章

## 二、每章写前必答六问

写任何一章之前，先回答这六个问题：

1. 这章推进的是哪一层主线目标？
2. 这章的核心冲突是什么？
3. 这章结束后，局面具体发生了什么变化？
4. 这章会让读者对谁产生新的认知？
5. 这章有没有新增物品、能力、规则或伏笔？
6. 这章结尾的钩子是什么？

六个问题答不出来，就不要急着写正文。

## 三、写作流程

### 写前

先看这些文件：

- 总纲
- 思维笔记
- 写作风格约束
- 时间线约束与时间轴
- POV视角约束
- 能力升级约束
- 分卷目标与高潮约束
- 人物台账
- 物品与能力台账

卷相关文件统一放在对应卷目录下，推荐固定结构：

- `卷XX_卷名/章纲`
- `卷XX_卷名/正文`
- `卷XX_卷名/章节记录`

然后填写一份“章节记录模板”，明确本章要做什么，并预估本章字数。

如果这章涉及尚未确定的设定、反转、关系推进方式，先在思维笔记里记录判断，再进入正文。

### 写中

正文创作时，优先保证三件事：

- 场景清晰
- 冲突明确
- 结尾有钩子

### 写后

正文写完后，必须同步更新：

- 本章实际字数
- 章节总表
- 思维笔记
- 时间线约束与时间轴
- 人物台账
- 关系台账
- 物品与能力台账
- 伏笔台账

如果本章涉及以下变化，还要额外同步：

- 切POV：更新POV视角约束中的记录表
- 能力成长：更新能力升级约束中的升级记录表
- 阶段推进：对照分卷目标与高潮约束，确认没有跑偏

### 卷末复盘

每写完一个阶段或10章，做一次复盘：

- 哪些角色变了
- 哪些关系变了
- 哪些能力升级了
- 哪些伏笔埋了还没回收
- 主线有没有被支线稀释

## 四、判定“写偏了”的五个信号

如果出现下面情况，说明这本书开始跑偏了：

1. 连续三章都没有明显推进主线
2. 主角一直在反应局面，而不是主动选择
3. 新能力越来越多，但代价越来越少
4. 配角很多，但记不住、分不清、功能重复
5. 前面埋过的重要信息，后面没人再提

## 五、推荐执行规则

为了让这套系统真正有用，建议固定执行以下规则：

1. 每写一章，先写章节记录，再写正文
2. 每写完一章，立刻更新台账，不要拖
3. 每10章做一次小复盘
4. 每一卷结束后做一次大复盘

## 六、当前阶段最重要的事

现阶段不要急着铺满后100章。

先把下面三件事做稳：

1. 第一卷目标和节奏
2. 前10章的抓力
3. 沈辞的能力首次亮相方式

只要这三件事稳住，后面往100章、300章、800章扩都会更顺。
