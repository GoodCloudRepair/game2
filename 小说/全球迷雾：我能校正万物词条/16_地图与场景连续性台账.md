# 地图与场景连续性台账

这份文档记录长篇中的地点、路线、空间层级和场景功能。  
它不追求画出精确地图，重点是防止路线、距离、空间归属和阶段气质写乱。

## 使用规则

1. 新地图或新大型场景第一次出现后，补入“地点总表”。
2. 每次跨场景转移，要能说明上一站、下一站、转移方式。
3. 如果一个地点同时有现实层和门域层，要分开记录。
4. 后续扩地图时，先查本表，避免新地点与旧路线冲突。
