# 角色退场与再登场看板

这份文档用于控制长篇群像的“谁还在场、谁暂时退场、谁后面必须回来”。  
它不替代人物台账，重点是防止配角消失无交代、重复登场无功能、或卷尾突然拉人救场。

## 使用规则

1. 重要角色连续离场超过10章，建议在本表标注当前去向。
2. 角色再登场前，至少确认其再登场功能：救场、冲突、信息、情感、代价。
3. 如果角色只为递一句信息而回来，优先考虑改成物件、记录、广播或其他在场角色完成。
4. 卷尾阶段不宜新拉大量旧角色，应优先回收已经有功能的人。

## 状态分类

| 状态 | 含义 | 使用建议 |
| --- | --- | --- |
| 在场 | 正在随主线行动或同场影响局面 | 需要持续给动作，不能只站着 |
| 外线在场 | 不与沈辞同场，但在外线推进封控、救援或信息 | 适合卷尾内外协同 |
| 暂退 | 阶段功能完成，短期不出现 | 需要记录最后状态 |
| 待回 | 已埋下再登场价值，后续应回收 | 再登场前先写清目的 |
| 封存 | 本卷不再使用，后续卷视情况再开 | 不要卷尾突然拉出 |

## 再登场前检查

- 这个角色回来是否改变局面？
- 读者是否还记得他，如果不记得，是否有自然提示？
- 再登场是否挤压当前章节核心冲突？
- 他这次回来，是完成自己的目标，还是只替作者传话？
- 如果他本章不回来，剧情是否仍能成立？如果能，是否真的需要他？
