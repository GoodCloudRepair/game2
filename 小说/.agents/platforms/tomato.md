# 平台插件：tomato（番茄小说）

评分口径：**番茄平台读者抓力 / 追读意愿**，不是文学完成度。

## 维度

| 维度 | 看什么 |
|------|--------|
| 吸引度 (0–10) | 前300字抓力、章内小爽点、规则/设定新鲜度、情绪钩、章末点击欲、信息密度负担（负担过高扣分） |
| 衔接度 (0–10) | 上章钩子是否被本章第一屏接住、换场/换目标是否顺、信息交接是否清楚、断章后是否还想点 |
| 番茄向判断 | 一句：本章为什么追 / 为什么掉 |

可选（若本书有全表）：千人留存预估 —— 仅作趋势，非后台真数。

## 交付时输出

```markdown
### 平台审（tomato）
- 吸引度：X.X
- 衔接度：X.X
- 判断：…
- 预警：（若某维明显偏低，列 1–3 条建议；不自动改文）
```

## 落盘

若本书存在 `13_番茄读者章节评分.md`（或等价全章评分表），写完一章**追加一行**；否则分数只出现在交付报告。

## 与底审关系

本插件**不替代**第10节连续性检查。底审不过 → 必须改文；本插件分低 → 只预警。
