# 长篇连载写作 · 轻量试验版（纲先行）

> 试验假设：**写章所需要素全在「10章章纲」里** → 填字时几乎不读台账。  
> **不替换** `novel-serial-writing`。需用户明确点名本 skill。

## 最简一行版

```
确认10章章纲存在（没有就先建）→ 本章写章卡是否写满（没满先补卡）→ 按卡填字 → 薄回填
```

---

## 核心流程（两段式）

### 段1：纲（重一点，但低频）

只在「缺这段10章纲 / 本章写章卡不达标」时做。一次做完 **10 章**，之后连写多章都吃同一份文件。

1. 定位目标章号 N → 找 `卷XX_…/章纲/卷XX_AAA-BBB章章纲.md`（含 N 的那段）
2. **没有** → 先创建该 10 章章纲（见下方「写章卡」模板，10 章都写满）
3. **有但本章条目太瘦**（缺场景节拍/在场人物/禁触等）→ **只补本章写章卡到达标**，不要顺手重写整段除非用户要求
4. 建/补纲时允许定点查台账（B/C）；建完后把结论写进卡里，避免写正文再查

### 段2：填字（极轻）

写章卡已达标后，写前 **只读**：

1. 本章写章卡（同一文件内本章那一节）
2. 上一章正文 **末约 40 行**（语气与未收钩）
3. （可选）`当前写章上下文包.md` 里「禁触 / 未收钩」——若写章卡已含则可跳过

然后：按卡里的节拍填到 2000–2200 字 → 薄回填。

**禁止**：填字阶段为「保险」通读 `02/03/05/08/12` 或五层总纲。

---

## 写章卡 = 大纲必须提供的全部要素

每章在 10 章章纲里用下面结构（可复制）。**缺任一项 = 不达标 = 不许开写正文。**

模板全文见同目录 [write-card-template.md](write-card-template.md)。

摘要清单：

| 区块 | 必须有 |
|------|--------|
| 定位 | 标题、功能、POV、时间、地点 |
| 六问 | 推进哪层、冲突、局面变化、新认知、唯一关键新信息、章末钩子 |
| 节拍 | 开场状态 + 事件1/2/3 + 结尾（每条一句，可直接扩写） |
| 在场 | 谁在场、各自一句行动；谁不在场 |
| 代价/规则 | 本章能力或手续怎么用、付什么代价 |
| 禁触 | 本章不能写破的 3–5 条 |
| 衔接 | 必须接住的上章钩；必须留给下章的钩 |
| 情绪/节奏 | 一句：偏冷/疼/压/反转等 |

现有书里偏瘦的章纲（只有功能/冲突/剧情/新信息/钩子）= **半成品**；lite 填字前要先补成写章卡。

## 与原版差异

| | 原版 | 本试验版 |
|--|------|----------|
| 写前准备 | 每章扫台账 | 先确认/创建10章写章卡 |
| 信息载体 | 台账 + 章节记录 | **章纲写章卡**为主 |
| 填字读入 | 多文件 | 本章卡 + 前章末 |
| 章节记录 | 写前完整填 | 可写后从卡**薄落地**（见下） |
| 台账 | 每章多文件必改 | 02+08 必追加；其余有变才改；可选刷上下文包 |

---

## 章节记录（薄落地）

仍建议落盘 `章节记录/第XXX章_章节记录.md`，但 **lite 允许**：

- 写前：不必先填完整 1–9 节；写章卡已含六问与节拍即视为「可写」
- 写后：用写章卡 + 实际正文，薄填记录（定位/六问/字数/第10节勾选），不重复造剧情

若用户坚持「先记录后正文」，把写章卡内容粘进记录第2–3节即可，仍不要另开台账大扫。

---

## 正文硬约束（填字时）

| 项 | 规则 |
|----|------|
| 字数 | ≥2000；软目标 2000–2200；必要时 ≤2300 |
| 新信息 | 严格只用写章卡里那 **1 条**关键新信息 |
| 解释句 | `明白了/看懂了/意识到/这说明` ≤1 |
| 禁触 | 写章卡「禁触」一条都不能破 |

保存：`卷XX_卷名/正文/第XXX章_标题.md`

---

## 薄回填（填字后）

详细规则见 [backfill.md](backfill.md)。摘要：

**每章必回（追加，不通读全文）**

- `02_章节总表.md` ← 写章卡：标题/局面/钩子/字数
- `08` 时间轴 ← 写章卡：时间/事件一句
- `当前写章上下文包.md` ← 刷新现在时/未收钩/禁触

**有变才回（写章卡「回填标记」为有才动）**

- `03/04/05/06`：Grep 人名或伏笔ID → 改一行或追加一行
- `14`–`20`：同理，无触发零读取

**禁止**：为回填通读大台账；禁止攒 10 章再回 `02/08`。

**每10章**：单独开「对账」会话（见 backfill.md），不在填字会话里做全量一致检查。

写章卡建议带：

```markdown
### 回填标记
- 03人物：无 / 有→…
- 04关系：无 / 有→…
- 05物品能力：无 / 有→…
- 06伏笔：无 / 新埋|触动|回收→…
- 14-20：无 / 有→…
```

## 建 10 章纲时（段1）允许多读一点

创建新一段 `卷XX_AAA-BBB章章纲.md` 时：

1. 读卷纲里「本阶段目标」短节（或阶段大纲开头）
2. 读上下文包或 `02` 最近 10 行，确认起点
3. 需要时 Grep `03/06` 补禁触与未收钩
4. **一次写出 10 张写章卡**（可略简，但每张仍含上表必填项）
5. 请用户确认这段纲后，再进入段2填字

用户只说「写第N章」且纲已达标 → 跳过段1，直接段2。

---

## 会话策略

- 段1（建10章纲）：单独一会话最好，产出只是章纲文件
- 段2（填字）：一会话 1–3 章；只带本章卡 + 前章末
- 回复末附：`[lite] 阶段：建纲|补卡|填字；写前读取文件数：X`
